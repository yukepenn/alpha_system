## Findings

### Warnings (non-blocking)

- **W1 (P194500): semantic change without contract change for
  `reset_on_session` features.** Returns / log-returns / rolling-volatility
  / ATR / volume-zscore / rolling-reduce features now derive session-reset
  boundaries from timestamps (previously canonical `session_label`, which
  was degenerate-static on real data → effectively never reset), but they
  are NOT in `_SESSION_TRUTH_FEATURES` (family.py:139-150), so their
  contract parameters and `feature_version_id`s are unchanged. Any
  previously materialized REGISTERED reference values for such features are
  now stale relative to current code at the same version id. Coordinator
  should audit the local registry for REGISTERED OHLCV reset_on_session
  features and deprecate/re-materialize, or extend the session-truth
  provenance parameters to these features in a follow-up.
- **W2 (P194500, handoff-disclosed): residual fast/reference divergence
  outside the vwap pack.** Non-vwap fast packs (regime, volume/activity,
  …) still key their internal session grouping off canonical
  `session_label`; on real data (static label) they would diverge from the
  repointed reference. Currently contained because (a) parity is enforced
  on timestamp-aligned fixtures and (b) the fast path is
  reference-parity-gated before materialization, but those packs must not
  be re-materialized on real data until they adopt the shared truth.
- **W3 (P200000, handoff-disclosed): report-surface semantics shift.**
  `registered_feature_records` / quality / coverage report sweeps no longer
  see deprecated rows. Correct per the function contract and currently
  caller-free, but any future "audit/history" report must use
  `resolve_feature(include_deprecated=True)`, not the guard view.
