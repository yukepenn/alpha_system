## Summary

Added the feature-layer contract family in
`alpha_system.features.contracts`. The new module defines immutable, hashable
contract objects for `FeatureSpec`, `FeatureFamily`, `FeatureInputSpec`,
`TransformSpec`, `WindowSpec`, `NormalizationSpec`, `FeatureSetSpec`,
`FeatureValueRecord`, `FeatureVersion`, and `FeatureLineageRecord`.

The contracts are metadata-only. They require `available_ts` on
`FeatureValueRecord`, reject centered/future/offline-only windows for live
`FeatureSpec`s, derive deterministic content-addressed `FeatureVersion` ids,
and require an FLF-P05 `FeatureRequestGateDecision` before a spec can be marked
`implementation_eligible`.
