## Summary

Added `alpha_system.features.request_gate`, a fail-closed feature-layer adapter
over existing governance `FeatureRequest` and duplicate-exposure guard objects.
The module validates requests through governance, invokes
`check_duplicate_exposure(...)`, applies guard notes through
`apply_duplicate_exposure_notes(...)`, and admits implementation only when the
checked request remains `APPROVED`.

The implementation includes:

- typed rejection reasons for missing, invalid, unapproved, duplicate-blocked,
  unavailable exposure-check, and invalid exposure-record outcomes;
- immutable `FeatureRequestGateDecision` and raising/predicate helper functions;
- read-only `DuplicateExposureReport` and `EquivalentFeatureGroup` views over
  governance `ExposureCheckResult` / `ExposureFinding` output;
- focused synthetic unit tests for missing request, invalid request, `PENDING`,
  `NEEDS_REVIEW`, `BLOCKED_DUPLICATE`, unavailable duplicate registry, duplicate
  blocking, and approved non-duplicate admission;
- durable gate documentation and a compact README snapshot update for FLF-P05.

No governance module was edited and no duplicate-exposure logic was
re-implemented.
