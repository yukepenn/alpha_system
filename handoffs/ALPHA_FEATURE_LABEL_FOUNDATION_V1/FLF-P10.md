## Summary

Implemented the additive Session / Calendar / Roll feature family under
`alpha_system.features.families.session`. The family consumes canonical OHLCV
input views plus explicit in-memory schedule / contract-definition metadata,
builds approved FLF-P06 `FeatureSpec` definitions through the FLF-P05
`FeatureRequest` gate, derives deterministic `FeatureVersion` records, and
returns in-memory `FeatureValueRecord` tuples only.

Covered FLF-P10 features:

- `session_id`
- `minutes_from_rth_open`
- `minutes_to_rth_close`
- `rth_segment_flag`
- `eth_segment_flag`
- `day_of_week`
- `roll_proximity_minutes`
- `expiration_proximity_minutes`
- `market_status`
- `halt_flag`

Each returned value carries the current canonical input row `available_ts`.
Schedule, roll, expiration, status, and halt context is treated as deterministic
metadata and is not derived from later trade or quote rows. Missing metadata
emits `None` with explicit unavailable flags rather than fabricated values.
