## Summary

Added the canonical feature/label input-view layer at
`alpha_system.features.input_views`.

The module:

- builds OHLCV and BBO views only from an `AcceptedDatasetVersion` handle plus
  in-memory canonical mappings through `alpha_system.features.consumption`;
- exposes frozen OHLCV/BBO row dataclasses with all five canonical timestamps;
- orders rows and `as_of(...)` usability strictly by `available_ts`;
- surfaces canonical BBO quote fields, optional quote fields, and BBO
  `quality_flags` tokens without imputation, filling, dropping, or policy
  interpretation;
- keeps raw provider-shaped fields out of the view surface by relying on the
  canonical `from_mapping` loaders to reject unsupported mappings.

Added unit and no-lookahead tests using tiny in-memory synthetic fixtures, added
the durable `INPUT_VIEWS.md` documentation, and updated the README snapshot for
FLF-P03 / FLF-P04.
