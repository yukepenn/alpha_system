# Specs

`specs/` holds the **phase-spec template only** — it is not a store of executed specs.

- `000-template.md` — the structure for a phase spec.
- Live phase specs are generated by Workflow 2 into
  `runs/<RUN_ID>/phases/<PHASE_ID>/spec.md` (local-only; never committed).
- Durable per-phase context is recorded as handoffs under `handoffs/` after execution.

Specs describe intended behavior and implementation plans before broad or risky
changes. See `docs/AGENT_CONTEXT_MAP.md` for where specs fit in the workflow.
